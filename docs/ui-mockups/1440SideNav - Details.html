Description

The primary vertical navigation for the application, providing access to all major sections. It can be expanded to show icons and text, or collapsed to show only icons.

Design System Alignment

Uses design system tokens for colors, spacing, and typography. It incorporates the Icon atomic component. It defines a key part of the application's main layout structure.

Responsive Specifications

Mobile Adaptations

  • The SideNav is permanently in its collapsed state below 1280px.
  • CSS tooltips are disabled in the collapsed state below 1280px as hover is unreliable on touch devices.

Tablet Adaptations

  • Same as mobile; the component is collapsed on all viewports below the 1280px desktop threshold.

Desktop Enhancements

  • Defaults to the expanded state on viewports >= 1280px.
  • Can be toggled to a collapsed state where icons remain and text labels appear in tooltips on hover.

Accessibility Features

Keyboard Support

The navigation is a list of links, fully navigable using the Tab key. Enter key activates the focused link.

Screen Reader Support

Uses a `

` landmark role with an `aria-label`. Navigation items are structured as an unordered list (`
    `). Links contain both an icon (hidden from screen readers with `aria-hidden`) and descriptive text.

Focus Management

A clear, high-contrast focus ring is applied to links using `:focus-visible`.

Aria Attributes

  • aria-label on the root `
    ` element.
  • aria-current='page' on the active link.
  • aria-hidden='true' on decorative icons.

Color Contrast

Text and icon colors meet WCAG 2.1 AA contrast requirements against the dark background.

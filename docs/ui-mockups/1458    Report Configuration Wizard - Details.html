Responsive Specifications

Desktop Layout

  • Centered card-based layout with a maximum width.
  • Horizontal stepper component at the top.
  • Two-column layouts for form fields where appropriate.

Tablet Adaptations

  • Switches to a vertical stepper layout to save horizontal space.
  • Form fields stack into a single column for better readability and touch interaction.

Mobile Adaptations

  • Follows the tablet adaptation with a single-column layout, ensuring all controls are easily tappable and readable on small screens.

Accessibility Features

Keyboard Navigation

Full keyboard-only navigation through all wizard steps, form fields, and action buttons is supported. Logical tab order is maintained.

Screen Reader Support

Semantic HTML structure with `form`, `section`, and `nav` landmarks. All form controls have associated labels. Stepper steps use ARIA attributes to announce the current, completed, and upcoming steps.

Focus Management

Focus is managed programmatically when moving between steps, typically landing on the first input field of the new step. Focus indicators are clear and visible.

Aria Attributes

  • aria-current='step'
  • aria-labelledby
  • aria-describedby
  • role='alert'

Color Contrast

All text, UI controls, and state indicators (e.g., validation errors) meet or exceed WCAG 2.1 AA contrast ratio requirements.

Responsive Specifications

Desktop Layout

  • Two-column (side-by-side) layout for the diff viewer, occupying the main content area.
  • Header contains metadata and a primary navigation action ('Back').
  • Optimized for 16:9 aspect ratio and wider.

Tablet Adaptations

  • Layout switches from side-by-side to a single-column, inline diff view at a breakpoint (e.g., 992px) to maintain readability.
  • Touch-friendly scroll targets.

Mobile Adaptations

  • Inline diff view is the default and only layout.
  • Header elements may stack or be simplified.
  • Full-width interactive elements.

Accessibility Features

Keyboard Navigation

Logical tab order from the 'Back' button to the scrollable diff panes.

Screen Reader Support

Semantic HTML for layout. Diff lines are marked up to be understandable (e.g., using hidden text or ARIA attributes to announce 'line added'/'line removed').

Focus Management

Clear focus indicators on the 'Back' button.

Aria Attributes

  • aria-label for diff lines
  • role='region' for diff panes

Color Contrast

Diff highlighting colors are chosen to meet WCAG 2.1 AA contrast ratios for graphical elements, and text contrast is maintained.

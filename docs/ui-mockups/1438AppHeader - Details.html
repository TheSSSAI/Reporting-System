Description

The main application header that provides branding, primary navigation controls, and user-specific actions like profile and notifications. It adapts responsively for smaller viewports.

Design System Alignment

Uses the design system's color palette for background and borders, spacing tokens for height and padding, and typography for the logo/title. It composes atomic Button and Icon components.

Responsive Specifications

Mobile Adaptations

  • On viewports < 1280px, the desktop utility navigation icons are hidden.
  • A hamburger menu toggle button is displayed to control the (off-canvas) SideNav.

Tablet Adaptations

  • The app title and user name next to the avatar become visible at 768px and wider.

Desktop Enhancements

  • Displays the full set of elements: brand, title, utility navigation, and user profile menu with name.

Accessibility Features

Keyboard Support

All interactive elements (buttons, user menu, dropdown links) are focusable and activatable via keyboard.

Screen Reader Support

The header is a landmark `

` element. The logo is a link with accessible text. Icon-only buttons have `aria-label` attributes. The user menu follows ARIA menu patterns.

Focus Management

Visible focus indicators are present on all interactive elements. Focus is managed within the dropdown menu.

Aria Attributes

  • aria-label for icon buttons
  • role="menu"
  • role="menuitem" for dropdown links

Color Contrast

All text and icons meet WCAG 2.1 AA contrast requirements.

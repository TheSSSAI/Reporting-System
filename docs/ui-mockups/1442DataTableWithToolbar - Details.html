Description

An organism component that combines a data table with a versatile toolbar for filtering, searching, and performing bulk actions on the table data. It manages different states based on user interaction, such as filtering and row selection.

Design System Alignment

This component orchestrates molecular and atomic components. It uses design system tokens for spacing, layout, and typography. The toolbar's responsive behavior and state transitions are key features.

Responsive Specifications

Mobile Adaptations

  • On screens < 900px, the toolbar stacks vertically.
  • The search bar takes full width on smaller screens.

Tablet Adaptations

  • The toolbar remains horizontal, but items may wrap if space is limited.

Desktop Enhancements

  • Full horizontal layout with primary actions on the left and secondary actions (like 'Create New') on the right.

Accessibility Features

Keyboard Support

All interactive elements in the toolbar (inputs, selects, buttons) are keyboard focusable and operable.

Screen Reader Support

The toolbar is contained within a `

` for semantic structure. The bulk actions state change is announced via an ARIA live region.

Focus Management

Logical focus order through toolbar elements and then into the data table.

Aria Attributes

  • aria-live='polite' for the bulk action selection count.
  • All dependencies (DataTable, Button, etc.) use their own required ARIA attributes.

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.

Implementation Notes

Embedded Css Architecture

All styles, including dependencies, are embedded in the HTML head. BEM is used for the organism (`data-table-organism__toolbar`) and its child molecular components.

Component Organization

The organism is structured with a `

` for the toolbar and a `
` for the data table content. State changes are controlled by a modifier class on the root element (`is-bulk-active`).

Responsive Design

A single media query is used to handle the main layout shift of the toolbar for smaller screens.

Accessibility Compliance

The component is structured for accessibility, relying on the a11y compliance of its child components and adding semantic structure at the organism level.

Browser Support

Uses modern CSS (flexbox, gap) supported by all target browsers.

Usage Guidelines

When To Use

This component is the primary interface for displaying and managing a collection of resources, such as user lists, report configurations, or job logs.

Content Guidelines

Toolbar actions should be labeled clearly and concisely. The search bar should have a descriptive placeholder.

Placement Rules

Typically serves as the main content for a list or management screen.

Customization Options

  • The contents of the primary and bulk action slots in the toolbar are fully customizable.
  • The DataTable component passed into the organism is configured separately.

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Edge.

Device Testing

Test responsive behavior at viewports above and below 900px.

Accessibility Testing

  • Verify keyboard navigation through the entire component.
  • Test with a screen reader to ensure the transition to bulk action mode is announced.
  • Check focus indicators on all toolbar elements.

N/A for the static structure, but the underlying DataTable's performance should be tested with large datasets.

Responsive Specifications

Desktop Layout

  • Primary three-pane layout with draggable dividers.
  • Header with title and primary/secondary actions.
  • Panes have minimum widths to prevent them from becoming unusable.

Tablet Adaptations

  • Layout is maintained, but touch targets for resize handles are larger.
  • Horizontal scrolling within panes may be necessary on smaller tablet viewports.

Mobile Adaptations

  • Not optimized for mobile. A message indicating that the feature is best used on a desktop may be displayed.

Accessibility Features

Keyboard Navigation

Logical tab order through all interactive elements: name input, save/cancel buttons, connector dropdown, preview buttons, and into the editor instances.

Screen Reader Support

All panes, buttons, and form elements have accessible labels. Dynamic content in the output pane (loading, success, error) is announced via ARIA live regions.

Focus Management

Focus is managed logically. The Monaco Editor component handles its internal focus. Focus is returned to the triggering element after actions.

Aria Attributes

  • aria-label
  • aria-describedby
  • role='alert'
  • aria-live

Color Contrast

All text, UI controls, and status indicators meet WCAG 2.1 AA contrast requirements.

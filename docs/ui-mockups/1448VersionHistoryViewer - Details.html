Description

An organism component, typically displayed in a modal, that shows a chronological list of saved versions for a configuration item. It allows for selection of versions to compare and provides actions to restore a previous version.

Design System Alignment

This component is composed of molecular and atomic components like ModalDialog, DataTable, Button, and Checkbox. It strictly adheres to the design system's tokens for typography, color, spacing, and interaction patterns.

Responsive Specifications

Mobile Adaptations

  • The modal becomes full-screen to maximize content area.
  • The table within the modal becomes horizontally scrollable to accommodate all columns without breaking the layout.

Tablet Adaptations

  • The modal remains a centered dialog.
  • The table remains horizontally scrollable.

Desktop Enhancements

  • The modal has a generous max-width to display table content clearly.
  • Hover states on table rows provide interaction feedback.

Accessibility Features

Keyboard Support

The modal traps focus. All interactive elements (close button, checkboxes, action buttons) are keyboard navigable and activatable (Tab, Space, Enter).

Screen Reader Support

The modal uses `role='dialog'` and `aria-modal='true'`. The table uses semantic HTML (`

`, ``, `
`, etc.) for proper screen reader announcement. Action buttons have descriptive `aria-label`s.

Focus Management

Visible focus rings are present on all interactive elements. Focus is programmatically managed when the modal opens and closes (requires JS).

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby
  • aria-label for checkboxes and buttons

Color Contrast

All text and UI elements meet or exceed WCAG 2.1 AA contrast requirements.

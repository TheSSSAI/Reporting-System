Description

A specialized three-pane layout for creating, editing, and live-previewing JavaScript transformation scripts. It integrates a code editor, a sample data input area, and an output/results panel.

Design System Alignment

This organism composes molecular and atomic components like Button, SelectDropdown, and Alert. It utilizes the design system's tokens for spacing, color, typography, and borders to create a cohesive and complex user interface.

Responsive Specifications

Mobile Adaptations

  • On viewports below 900px, the layout switches from a two-column grid to a single-column, three-row grid to stack the panes vertically for readability.

Tablet Adaptations

  • The layout remains a two-column grid but the relative sizes of the panes can be adjusted by the user. Touch targets for resizing would need to be larger.

Desktop Enhancements

  • The three-pane layout is optimized for wide screens, allowing all information to be visible simultaneously. Resizable panes allow users to customize their workspace.

Accessibility Features

Keyboard Support

All interactive controls (buttons, selects) are keyboard focusable and operable. Focus management for pane resizing would require JavaScript.

Screen Reader Support

Panes are structured with semantic headers (`

`) for easy navigation. State changes (loading, success, error) are announced via ARIA live regions for the Alert component.

Focus Management

Focus indicators are provided for all interactive controls. In a real implementation, focus should be managed when opening modals or triggering actions.

Aria Attributes

  • role='alert' for the error alert component.
  • aria-label for buttons and other controls to provide context.

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.

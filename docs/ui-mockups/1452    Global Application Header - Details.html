Responsive Specifications

Desktop Layout

  • Full horizontal layout displaying the logo, all primary navigation links, and user action icons.
  • Max-width container to ensure readability on very wide screens.

Tablet Adaptations

  • Primary navigation links collapse into a hamburger menu to save space.
  • Logo and action icons remain visible.

Mobile Adaptations

  • Header height may be slightly reduced.
  • All navigation is contained within a hamburger menu.
  • Touch target sizes are maintained for all interactive elements.

Accessibility Features

Keyboard Navigation

Logical tab order from left to right: Logo, Navigation Links, Action Icons, User Profile. Dropdown menus are fully navigable.

Screen Reader Support

and
landmarks. Buttons with only icons have descriptive aria-labels.

Focus Management

Clear, visible focus indicators on all interactive elements. Focus is managed correctly when opening/closing menus.

Aria Attributes

  • aria-label for icon buttons
  • aria-current='page' for the active navigation link
  • aria-expanded and aria-controls for dropdowns/menus

Color Contrast

All text, icons, and focus indicators meet WCAG 2.1 AA contrast requirements against their backgrounds.
